# LINE Management アプリケーション 総合解析レポート

本ドキュメントは、Supabase Edge Functions と LINE Messaging API、Google Workspace (Calendar/Gmail) 、そして Groq API（LLM）を活用して構築された「LINE Management」アプリケーションの全機能を網羅的に解析したものです。

## 1. アプリケーション概要

LINEのグループ/ルームにおける会話を自動で取り込み、AIを活用した「要約配信」「予定の自動登録・照会」「過去会話の検索」、そして「Gmailの予約メール通知」などを行う高度な運用ボットアプリです。
静的HTMLによる専用の管理画面（Admin Dashboard）を備えており、ルームごとの細かな機能制御や配信スケジュールを動的に管理できます。

---

## 2. コア機能（機能要件）

### 2.1. LINE メッセージの受信とAI自動解釈 (Webhook)
- **AIによる意図解釈**: LINEでメッセージが送信されると、Groq APIを用いてユーザーの意図を自動判定します（`create_calendar`, `list_calendar`, `search_messages`, `none`）。
- **確認プロセス（フォールバック）**: 信頼度がやや低い場合は、ユーザーに「予定を登録しますか？」といった形での確認（Pending Confirmation）を返し、誤登録を防ぎます。
- **コマンド応答**: AIだけでなく、明示的なコマンド（例: `予定登録 2026-05-07 14:30 60 5月会議`）による確実な操作もサポートされています。

### 2.2. 定期要約配信 (Summary Cron)
- **定時要約**: 指定された時刻（例: `12, 17, 23`時）に、前回配信から蓄積されたメッセージをAIが要約し、LINEに自動配信します。
- **集計方式のカスタマイズ**:
  - **各回独立**: 配信時刻ごとに独立して要約を生成。
  - **1日まとめ**: 毎回の配信は行わず、1日の最終配信時刻にのみ、その日の全会話をまとめて要約するモード。
- **処理タイミング**: メッセージを「処理済み」とするタイミングを、「配信成功ごと」または「1日の終わり」から選択・設定可能です。

### 2.3. Googleカレンダー連携 (Calendar Integration)
- **AI予定自動登録**: 会話の中に「明日の14時から新宿でミーティング」等の情報が含まれると、AIが日時・場所・タイトルを抽出し、Googleカレンダーへ自動登録します。
- **予定の照会**: 「今日の予定を教えて」といった質問に対し、該当期間のカレンダーイベントを検索して LINE 上で返答します。
- **翌日予定のリマインダー**: 指定した時刻に、明日予定されているイベントを自動通知します。「予定がない日は通知しない」などの設定も可能です。

### 2.4. 会話の保持と検索機能 (Message Search)
- **履歴の保持期間制限**: プライバシーとストレージ容量の観点から、メッセージの保持期間を `60日 / 120日 / 180日` から選択可能です。
- **AI会話検索**: 「1週間前の会議で何が決まったか教えて」といった問いかけに対し、DB内の過去メッセージを検索・フィルタリングし、要約して回答します。

### 2.5. Gmail 予約通知機能 (Gmail Alert)
- **メール連携**: 指定されたGmailアカウントを定期的にポーリング（`gmail-alert-cron` またはリアルタイム通知）し、予約完了メールなどを検知します。
- **AIメール要約**: 必要な場合、メールの本文ごとAIで要約し、対象のLINEルームへと即座にアラートとして転送します。

### 2.6. メディア管理 (LINE Media Storage)
- **メディアの自動保存**: LINEに投稿された 画像・動画・音声・ファイル を Supabase Storage に自動アップロードして保存します。
- **容量制限とクリーンアップ**:
  - アップロードの最大サイズ設定（例: 10MB/20MB）。
  - ストレージ上限（500MBキャパシティ等）によるフェイルセーフ機能。
- **専用ビューワー**: `media.html` 経由で、過去に共有されたメディアコンテンツをブラウザから閲覧・管理できます。

## 3. 管理画面（Admin Dashboard: `index.html`）

管理画面はトークン認証（`ADMIN_DASHBOARD_TOKEN`）によって保護されており、以下の操作をGUIでリアルタイムに実行・管理できます。

1. **全体設定 (Global Settings)**
   - アプリ全体の有効化/無効化
   - 定期配信の時刻設定、最終集計モード、メッセージ保持期間の設定。
   - ストレージ利用状況のモニタリング（DB・メディア両方）。
2. **ルーム別詳細設定 (Room-Specific Configurations)**
   機能ごとにルーム（グループ）単位でON/OFFを細かく切り替えられます。これにより「雑談ルームでは自動応答をオフ」「特定ルームのみ要約を配信」という運用が可能です。
   - **有効/無効**: ルーム自体の処理対象フラグ。
   - **ルーム要約配信**: 個別の要約を配信するか。
   - **明日予定配信**: リマインダーをこのルームで行うか。
   - **会話検索応答**: 過去会話の検索を許可するか。
   - **自動応答**: AIの自動返信（予定抽出など）を許可するか。
   - **自動登録**: AI判断によるカレンダー登録を許可するか。
   - **Gmail予約通知**: このルームへGmailアラートを流すか。
3. **配信ログ (Delivery Logs)**
   - スケジュールごとの成功・失敗、対象件数などのログを履歴として確認・追跡可能です。

## 4. アーキテクチャと外部連携サービス

| コンポーネント / サービス | 役割 |
| :--- | :--- |
| **Supabase (PostgreSQL)** | 全データ（メッセージ, 設定, ログ）の保存と、Edge Functionsによるバックエンド処理。`pg_cron`等を利用したデータベースレベルのバッチ処理も担う。 |
| **Supabase Storage** | メディア（画像、動画など）を保存するオブジェクトストレージ。 |
| **LINE Messaging API** | Webhookのエンドポイントとして機能し、ユーザーからのメッセージ受信・返信・プッシュ通知を行う。 |
| **Google Calendar API** | サービスアカウントを介したイベントの作成・取得。 |
| **Gmail API** | `refresh_token` を用いて、新着予約メールの取得・読み取りを行う。 |
| **Groq API** | 高速なLLM推論API。意図解釈、テキスト要約、日時・場所の抽出、過去会話からの回答生成を行う。 |

---

## 5. デプロイ構成ディレクトリ

- `supabase/functions/line-webhook/`: 受信、解釈、カレンダー処理、メディア保存など、メインとなるインタラクティブ処理。
- `supabase/functions/summary-cron/`: 定期的な要約配信と、翌日カレンダー予定のプッシュ通知を処理。
- `supabase/functions/gmail-alert-cron/`: GmailAPIを監視しアラートをLINEに流すバッチ処理。
- `supabase/functions/admin-api/`: 管理ダッシュボード(`index.html`)へのREST API提供（認証・設定反映）。
- `supabase/migrations/`: テーブル定義や `pg_cron` セットアップなど、高度な状態管理・履歴管理が行われている SQL ファイル群。
- `index.html` & `media.html`: 管理用フロントエンド（SPAライクに動作）。

## 6. 特筆すべき考慮点・セーフティネット

- **AIの暴走防止と確認プロセス**: カレンダー登録においては、信頼度（Confidence）の閾値が設定されており、曖昧な指示にはアクションを起こさず確認メッセージを返します。
- **コストとストレージ保護**: メッセージの定期的クリーンアップ (60/120/180日保証) や、メディアファイルのアップロード上限・総量上限が組み込まれており、DB容量の枯渇を防ぐ運用をサポート。
- **エラーハンドリング**: スケジュール外の配信制御、APIキー未設定時のグレースフル・プロテクション、送信失敗等の各種エラー詳細がログに記録され管理画面から追跡できるよう設計されています。
